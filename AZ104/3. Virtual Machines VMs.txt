# AZ VM 
- Shared hypervisors (multi-tenant)
- Dedicated hosts (premium)
- Creating a Virtual machine also create following resources: Public IP address, Network security group, Network Interface, Disk, Virtual network...
- AZ VM Management: AZ portal, AZ (CLI), PowerShell, APIs templates
# Supported OS for VMs:
- Windows Server & Client versions
- Linux distributions (Red Hat, Ubuntu, CentOS, etc.) - Required Port TCP 22, when RDP is port 3389
# AZ VM Details
- Resource group, location, VM name
- Image, size (virtual CPUs), Random Access Memory (RAM), number of data disks
- OS credentials
- VM extensions
- Vnet, Subnet, public IP address (default enabled, but not recommended), Network Security Group (NSG)
- Security group, load balancing
- Auto-shutdown, backup

# AZ Windows VM
- Specify logon credentials
- Connect using Remote Desktop Protocol (RDP port 3389)
- Ensure firewalls allow RDP
- Remote management through Azure Bastion: No public IP required
- You can then save it as a virtual machine image to deploy other VMs from it.

# AZ Linux VM (TCP 22)
- Username/password authentication
- Public key authentication
- Connect using Secure Shell (SSH)
- Ensure firewalls allow SSH
- Remote management through Azure Bastion: No public IP required 

# AZ VM Scale Set (VMSS)
- Used with load balancing
- The scale set can be autoscaled: Scaling out,Scaling in
- VMs must be identical: Same OS image
- Spot instances: Reduce costs

az rest --method get --url /subscriptions/{subscriptionId}/resourceGroups/{resourceGroupName}/providers/Microsoft.Compute/virtualMachines/{vmName}?api-version=2019-03-01
# Create a default Ubuntu VM with SSH authentication
az vm create --name MyVm --resource-group MyResourceGroup --image UbuntuLTS
# List all VM images in table format
az vm image list --output table
# List VM images for Windows Server in East US region
az vm image list --location eastus --publisher Microsoft --offer WindowsServer --all --output table
# List VM images for Windows Server 2022 in East US region
az vm image list --location eastus --publisher Microsoft --offer WindowsServer --sku 2022 --all --output table
# Create a VM with specific configuration (Windows Server 2022, subnet, vnet, public IP, admin credentials)

az vm list --query [].name

# Useful cmdlets for VM configuration:
# Add-AzVMNetworkInterface -> Set network profile
# Set-AzVMOperatingSystem -> Set OS profile
# Set-AzVMSourceImage -> Set source image
# Set-AzVMOSDisk -> Set OS disk (storage profile)
# Get-AzComputeResourceSku -> Find available VM sizes for subscription and region
# Prompt for credentials and store in variable
$creds = Get-Credential
# Create a new VM with specified configuration
New-AzVm -ResourceGroupName "Hq" `
         -Name "WinSrv2022-1" `
         -Location "EastUS" `
         -VirtualNetworkName "vnet1" `
         -SubnetName "subnet2" `
         -SecurityGroupName "Windows_Management_Rules" `
         -Image "MicrosoftWindowsServer:WindowsServer:2022-datacenter-azure-edition:latest" `
         -OpenPorts 3389 `
         -Credential $creds
# Get all VMs in a specific resource group
get-azvm -ResourceGroupName "Hq"

# Resize VM (Scale Up and Down), Vertical autoscaling (Uo = more resources, down = less resources)
> VM > Overview > Platform metrics (vCPUs, RAM (GiB), Data disks, Max IOPS, Local storage (GiB), Premium disk, Cost/month)
> VM > Availability + Scale > Size, select another size to change
get-command *resize*            
# find commands related to "resize"
$myvm = get-azvm -resourcegroupname hq -vmname WinSrv2022-1   # get VM object
$myvm | get-member -type property                            # list VM properties
$myvm.hardwareprofile | get-member -type property            # list hardware profile properties
# Resize the VM
Resize-AzVM -ResourceGroupName "rg1" -Name "vm1" -VMSize "Standard_D2s_v3"
# Start the VM
Start-AzVM -ResourceGroupName "rg1" -Name "vm1"

# Bastion connection creates a host on a subnet in virtual network
- vnet-bastion is a resource to connect to VM via browser (Virtual network subnet and vnet public address)
- It does not expose public IP of VM
- Connecting to VM via RDP (Remote desktop protocol) cant be used on internet as RDP has private key
- Internet needs public key to connect to VM 
> VM > Overview > Connect via bastion (Auth Type, Username,VM Password)
Manage VMs using Azure Bastion (private IP access).
- reset password > VM > Help > reset password


# Linux SSH public key authentication (Using .ppk private key via TCP 22)
- Public on cloud, private on device of user
> VM (Has to be running) > Help > Reset Password, Add SSH public key (Username, SSH public key source, key pair name) > Download User private key and convert to .ppk format (Use PuTTY key generator importing OpenSSH private key file, converting and saving as .ppk file) > Set passphrase > Paste the IP address on PuTTY key generator > Set port 22 > Specify path of .ppk > Connect to VM with user and passphrase

# Security Best Practices
Avoid public IPs for VMs.
Use Azure Bastion or VPN for private IP access.
Enable SSH key authentication for Linux.
Encrypt disks with customer-managed keys.
