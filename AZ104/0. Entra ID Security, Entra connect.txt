# CLoud shell CLI commands
# List groups
az ad group list --query [].displayName
# Creating group with mandatory fields (display name, nickname)
az ad group create --display-name ProjectA_Members --mail-nickname projecta_members
# Add user by object ID
az ad group member add --group ProjectA_Members --member-id 4518d5c8-2f53-415a-ac65-3836b4d110ee
# Show users on the group selecting display name
az ad group member add --group ProjectA_Members --query [].displayName
# Delete group
az ad group delete --group ProjectA_Members

# Entra ID Dynamic Groups (Different from traditional)
- Group determined by attribute (User or device attributes)
- require at least Entra ID Premium P1 licence or equivalent (E3), P2 (E5) also works 
- User and device attributes must be populated in Entra ID
- Membership rules builded using the Rule Builder GUI or (Can be edited by the expression sintax
- Set Owner of the group
- Add dynamic query

# New group (Group type, Group name, Membership type = DYNAMIC) If dynamic does not appear it might be for licence insufficiency
- Config rules with AND conditions or syntax (user.city -eq "Milan") and (user.usageLocation -eq "Milan")
user.accountEnabled -eq true
(user.department -eq null)
user.department -in ["A01","A02","A03"]
user.displayName -match "Da.*"
(user.department -eq "Sales") -and -not (user.jobTitle -contains "Manager")
(device.deviceOSType -eq "iPad") -or (device.deviceOSType -eq "iPhone")
(device.deviceOSType -contains "AndroidEnterprise")
